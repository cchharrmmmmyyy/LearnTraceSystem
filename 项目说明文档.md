## 一、项目概述

LearnTraceSystem 是一个用于采集和分析用户阅读行为数据的系统。系统通过采集用户在阅读英文文章时的鼠标悬停行为，分析用户的阅读习惯和学习行为。

### 核心功能流程

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│   前端页面       │────▶│    后端 API       │────▶│   行为分析       │
│  (数据采集)      │     │   (数据接收)       │     │   (数据处理)     │
└─────────────────┘     └──────────────────┘     └─────────────────┘
                                                        │
                                                        ▼
                                               ┌─────────────────┐
                                               │   结果展示       │
                                               │   (前端页面)     │
                                               └─────────────────┘
```

## 三、接口定义（当前版本）

### 3.1 行为数据采集接口

**接口地址**: `POST /api/behavior/collect`

**请求参数**:

| 参数        | 类型     | 必填 | 说明            |
| --------- | ------ | -- | ------------- |
| sessionId | string | 是  | 会话ID（URL查询参数） |
| body      | array  | 是  | 行为数据数组        |

**请求体示例**:

```json
[
    {
        "event_type": "hover",
        "paragraph_id": 1,
        "sentence_id": "1-1",
        "word_id": "1-1-3",
        "start_time": 1774367404011,
        "duration": 536
    }
]
```

**响应示例**:

```json
{
    "code": 200,
    "message": "success",
    "data": "行为数据采集成功！session=session_xxx，共 1 条"
}
```

### 3.2 行为识别接口

**接口地址**: `POST /api/behavior/recognize`

**请求参数**:

| 参数               | 类型     | 必填 | 说明            |
| ---------------- | ------ | -- | ------------- |
| sessionId        | string | 是  | 会话ID（URL查询参数） |
| answerSentenceId | string | 否  | 正确答案句子ID      |

**响应示例**:

```json
{
    "readingEfficiency": "high",
    "readingHabits": ["顺序阅读", "阅读顺序正确"],
    "identifiedProblems": []
}
```

### 3.3 行为分析接口

**接口地址**: `POST /api/behavior/analyze`

**请求参数**:

| 参数               | 类型     | 必填 | 说明            |
| ---------------- | ------ | -- | ------------- |
| sessionId        | string | 是  | 会话ID（URL查询参数） |
| answerSentenceId | string | 否  | 正确答案句子ID      |

**响应示例**:

```json
{
    "totalReadingTime": 5000,
    "sentenceDwellTimes": {"1-1": 3000, "1-2": 2000},
    "paragraphDwellTimes": {"1": 5000},
    "wordDwellTimes": {"1-1-1": 500, "1-1-2": 800},
    "backReadCount": 0,
    "readingPath": ["1-1", "1-2"],
    "positioningAccurate": false,
    "maxDwellSentence": "1-1"
}
```

## 四、数据模型

### 4.1 阅读事件 (ReadingEvent)

```json
{
    "event_type": "hover",
    "paragraph_id": 1,
    "sentence_id": "1-1",
    "word_id": "1-1-3",
    "start_time": 1774367404011,
    "duration": 536
}
```

| 字段            | 类型     | 必填 | 说明                                   |
| ------------- | ------ | -- | ------------------------------------ |
| event\_type   | string | 是  | 事件类型，固定值 `"hover"`                   |
| paragraph\_id | int    | 是  | 段落ID，从1开始                            |
| sentence\_id  | string | 是  | 句子ID，格式：`段落ID-句子ID`，如 `"1-1"`        |
| word\_id      | string | 是  | 单词ID，格式：`段落ID-句子ID-单词ID`，如 `"1-1-3"` |
| start\_time   | long   | 是  | 进入该单词的时间戳（毫秒）                        |
| duration      | long   | 是  | 停留时间（毫秒）                             |

## 五、当前问题记录

### 5.1 前端问题

#### 问题 1: 页面响应式布局不灵活

- **描述**: 窗口大小改变时页面布局会乱
- **影响**: 用户体验不佳，无法在不同尺寸的设备上正常使用
- **优先级**: 高

#### 问题 2: 文章内容写死

- **描述**: 前端的阅读文章内容是硬编码在代码中的
- **文件**: `src/scripts/pages/AnalyticManager.js`
- **影响**: 无法动态加载不同的阅读材料
- **优先级**: 高

#### 问题 3: 缺少结束信号

- **描述**: 前端在结束阅读时没有发送结束信号给后端
- **影响**: 后端无法判断会话何时结束，无法进行会话维度的完整分析
- **优先级**: 高

#### 问题 4: 未实现的功能

- **描述**: 以下功能尚未实现：
  - 阅读进度显示
  - 阅读结果展示
  - 历史记录查看
  - 数据导出功能（部分）
- **优先级**: 中

### 5.2 后端问题

#### 问题 1: 数据仅存储在内存

- **描述**: 后端仅将数据存储在 Java 内存变量中
- **位置**: `BehaviorService.sessionEvents`
- **影响**:
  - 后端重启后数据丢失
  - 无法进行历史数据分析
- **优先级**: 高
- **预计方案**: 先写入 TXT 文件，后续可考虑数据库

#### 问题 2: 缺少文章管理接口

- **描述**: 没有提供文章上传、存储、发送给前端的接口
- **影响**: 无法支持动态文章内容
- **优先级**: 高

#### 问题 3: 终端输出不规范

- **描述**: System.out.println 输出格式不统一，缺乏日志级别管理
- **影响**: 难以进行问题排查和日志分析
- **优先级**: 中

#### 问题 4: 分析程序未被调用

- **描述**: `analyzeBehavior` 和 `recognizeBehavior` 方法存在但未被调用
- **影响**: 采集的数据没有进行分析，用户无法获得行为分析结果
- **优先级**: 高

#### 问题 5: 缺少停止信号接口

- **描述**: 没有提供会话结束/停止的接口
- **影响**: 无法完整标记一个阅读会话的结束
- **优先级**: 高

### 5.3 接口问题

#### 问题 1: 分析接口定义不够清晰

- **描述**:
  - `analyze` 和 `recognize` 接口的区别不明确
  - `answerSentenceId` 参数的作用和必要性不清晰
  - 响应数据格式没有完整的文档说明
- **优先级**: 中
