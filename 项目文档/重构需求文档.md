- **编写日期**: 2026-03-24

---

## 1. 需求背景

原有的前端代码存在结构混乱、耦合严重、难以维护等问题：
- HTML、CSS、JavaScript 代码混杂在少量文件中
- 存在大量行内样式和行内事件处理
- 配置信息硬编码，分散在各处
- 模块职责不清，互相依赖
- 目录结构不清晰，扩展困难

## 2. 重构目标

按照**工程化、可扩展、高内聚低耦合**的标准，对前端代码进行彻底重构。

## 3. 重构要求

### 3.1 结构彻底分离

**要求**：
- 把 HTML、CSS、JavaScript 完全拆分成独立文件
- HTML 只保留结构，删除所有行内样式、行内事件
- CSS 单独文件，使用语义化 class，不写死 style
- JS 单独文件，交互、逻辑、渲染分离

**实现方式**：
- HTML 文件只包含语义化标签和必要的 id/class
- 所有样式移到独立的 `.css` 文件
- 所有 JavaScript 逻辑移到独立的 `.js` 文件
- 使用事件委托或外部事件绑定

### 3.2 消除硬编码

**要求**：
- 所有文字、标题、按钮文本、提示语抽成配置变量
- 颜色、尺寸、间距统一使用 CSS 变量
- 路径、链接、状态文本全部可配置

**实现方式**：
- 创建 `app.config.js` 配置文件
- 创建 `variables.css` CSS 变量文件
- 所有可配置项通过变量引用

### 3.3 代码模块化、高内聚低耦合

**要求**：
- 一个函数只做一件事
- 功能按模块拆分，不耦合、不混乱
- 逻辑清晰，不嵌套过深，不写冗余代码

**实现方式**：
- 按功能划分模块（数据采集、存储、UI、工具等）
- 每个模块独立文件，职责单一
- 模块间通过标准接口通信，避免直接依赖

### 3.4 整理目录结构

**要求**：
- 按页面/功能创建清晰文件夹
- 样式、脚本、图片分类存放
- 结构一目了然，方便后续扩展

**实现方式**：
```
LearnTraceSystem/
├── index.html              # 页面文件
├── src/
│   ├── config/            # 配置
│   ├── styles/            # 样式
│   ├── scripts/           # 脚本
│   │   ├── core/         # 核心模块
│   │   ├── modules/      # 功能模块
│   │   ├── pages/        # 页面逻辑
│   │   └── utils/        # 工具函数
│   └── components/        # HTML组件
```

### 3.5 完善注释与文档

**要求**：
- 每个文件顶部标注：功能、更新时间
- 关键逻辑必须加注释
- 自动生成或更新 README.md，说明：
  - 项目结构
  - 当前进度
  - 文件作用
  - 后续扩展方式

**实现方式**：
- 文件头注释包含：功能描述、更新时间
- 复杂逻辑添加行内注释
- 维护项目 README.md 文档

### 3.6 保证后续可无限扩展

**要求**：
- 新增功能时不影响原有代码
- 风格统一、易于维护
- 符合长期迭代、逐步开发的需求

**实现方式**：
- 遵循开放封闭原则（对扩展开放，对修改封闭）
- 保持统一的编码风格和命名规范
- 使用配置驱动，避免硬编码

## 5. 扩展指南

### 5.1 添加新页面

1. 在 `src/styles/pages/` 创建页面样式文件
2. 在 `src/scripts/pages/` 创建页面逻辑文件
3. 在 `src/components/` 添加需要的组件 HTML
4. 在 `app.config.js` 添加页面文本配置
5. 在 `main.css` 导入页面样式

### 5.2 添加新组件

1. 在 `src/components/` 创建 `.html` 文件
2. 在 `src/styles/components/` 创建样式文件
3. 在 `main.css` 中导入样式文件
4. 使用 `loadComponent()` 函数加载组件

### 5.3 修改主题

在 `src/styles/variables.css` 中修改 CSS 变量，所有样式会自动更新。

### 5.4 添加新的数据类型

1. 在 `app.config.js` 中定义数据格式
2. 在 `DataCollector.js` 中实现采集逻辑
3. 在 `DataStorage.js` 中添加存储方法
