## 二、数据结构定义

### 2.1 悬停数据记录（HoverRecord）

用户鼠标悬停在单词上的行为数据。

```json
{
    "event_type": "hover",
    "word_id": "1-3-5",
    "start_time": 1710000000000,
    "duration": 1200
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| event_type | string | 是 | 事件类型，固定值 `"hover"` |
| word_id | string | 是 | 单词ID，格式：`段落ID-句子ID-单词ID`，如 `"1-3-5"` 表示第1段第3句第5个单词 |
| start_time | number | 是 | 进入该单词的时间戳（毫秒） |
| duration | number | 是 | 停留时间（毫秒） |

### 2.2 阅读会话数据（ReadingSession）

一次完整的阅读会话信息。

```json
{
    "date": "2026-03-24T10:00:00.000Z",
    "duration": 120,
    "sentences_read": 5,
    "total_sentences": 20
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| date | string | 是 | 会话开始时间（ISO 8601格式） |
| duration | number | 是 | 阅读时长（秒） |
| sentences_read | number | 是 | 已阅读的句子数 |
| total_sentences | number | 是 | 总句子数 |

### 2.3 用户数据（UserData）

```json
{
    "id": "user_001",
    "name": "张三",
    "email": "zhangsan@example.com"
}
```

### 2.4 通用响应格式（Response）

```json
{
    "code": 200,
    "message": "success",
    "data": {}
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | number | 状态码，200表示成功，其他表示错误 |
| message | string | 响应信息 |
| data | object | 响应数据 |

## 三、API 接口列表

### 3.1 悬停数据接口

#### POST /api/hover-data

保存悬停数据到服务器。

**请求体**：
```json
{
    "session_id": "session_171000_abc123",
    "data": [
        {
            "event_type": "hover",
            "word_id": "1-3-5",
            "start_time": 1710000000000,
            "duration": 1200
        }
    ],
    "timestamp": "2026-03-24T10:00:00.000Z"
}
```

**响应示例**：
```json
{
    "code": 200,
    "message": "success",
    "data": {
        "id": "record_001"
    }
}
```

---

#### GET /api/hover-data/{sessionId}

获取指定会话的悬停数据。

**路径参数**：
| 参数 | 类型 | 说明 |
|------|------|------|
| sessionId | string | 会话ID |

**响应示例**：
```json
{
    "code": 200,
    "message": "success",
    "data": {
        "session_id": "session_171000_abc123",
        "data": [
            {
                "event_type": "hover",
                "word_id": "1-3-5",
                "start_time": 1710000000000,
                "duration": 1200
            }
        ]
    }
}
```

---

#### DELETE /api/hover-data/{sessionId}

删除指定会话的悬停数据。

**路径参数**：
| 参数 | 类型 | 说明 |
|------|------|------|
| sessionId | string | 会话ID |

**响应示例**：
```json
{
    "code": 200,
    "message": "success",
    "data": null
}
```

---

## 六、注意事项

1. 所有时间戳均使用毫秒级时间戳或 ISO 8601 格式
2. word_id 格式：`段落ID-句子ID-单词ID`，从1开始计数
3. 悬停数据最小采集时长为 60 毫秒（可通过配置修改）
4. 前端优先使用 localStorage 存储，离线可用
5. 后端对接后，数据会自动同步到服务器
