# LearnTraceSystem 最小可运行系统开发计划

## 1. 项目概述
本项目旨在开发一个学习行为分析与复盘系统的最小可运行版本，重点实现阅读采集前端功能。根据用户需求规格说明书，系统需要记录用户在阅读过程中的行为数据（页面滚动、鼠标停留、文本选择、标注操作等），为后续的行为分析和复盘报告生成提供数据基础。

## 2. 目标
开发一个最小可运行的系统原型，包含以下核心功能：
- 提供英文阅读材料界面
- 实时采集用户阅读行为数据
- 将采集的数据存储到本地（便于后续分析）
- 提供基本的用户控制界面

## 3. 技术栈选择
- **前端**: 纯 HTML5 + CSS3 + JavaScript (ES6+)
- **数据存储**: 浏览器本地存储 (localStorage) + 控制台输出
- **开发工具**: 无构建工具，直接浏览器运行
- **兼容性**: 支持 Chrome、Edge 等现代浏览器

## 4. 功能需求详述

### 4.1 阅读界面功能
- 显示一篇英语四级阅读文章（固定内容）
- 文章分段落显示，支持文本选择
- 页面支持正常滚动
- 简洁的视觉设计，不干扰阅读

### 4.2 行为数据采集功能
根据需求规格，需要采集以下行为数据：

| 行为类型 | 采集内容 | 采集方式 |
|---------|---------|---------|
| 页面滚动 | 滚动位置、滚动时间、滚动方向 | `scroll` 事件监听 |
| 鼠标移动 | 鼠标坐标、停留时间、移动轨迹 | `mousemove` 事件采样 |
| 文本选择 | 选中文本内容、起止位置、选择时间 | `selectionchange` 事件 |
| 标注操作 | 点击位置、标注类型、标注内容 | `click` 事件 + 自定义标注UI |
| 答题行为 | 选项选择、答题时间、修改记录 | 选择题UI交互 |

**最小可行集**: 先实现滚动、鼠标移动、文本选择三种核心行为采集。

### 4.3 数据存储功能
- 行为数据格式标准化（JSON格式）
- 实时存储到 `localStorage`
- 支持数据导出（JSON文件下载）
- 控制台实时显示采集数据

### 4.4 用户控制功能
- 开始/停止数据采集按钮
- 重置数据按钮
- 数据导出按钮
- 采集状态显示

## 5. 系统架构设计

### 5.1 文件结构
```
LearnTraceSystem/
├── index.html              # 主阅读界面
├── reading.html           # 纯阅读界面（可选）
├── styles.css             # 样式文件
├── app.js                 # 主应用逻辑
├── dataCollector.js       # 数据采集模块
├── dataStorage.js         # 数据存储模块
├── uiController.js        # UI控制模块
├── sample-data/           # 示例数据
│   ├── article.json       # 阅读文章数据
│   └── behavior-rules.json # 行为分析规则
└── assets/                # 静态资源
    └── images/
```

### 5.2 模块设计
1. **数据采集模块 (DataCollector)**
   - 初始化事件监听器
   - 行为数据标准化
   - 节流控制（避免过多数据）

2. **数据存储模块 (DataStorage)**
   - localStorage 操作封装
   - 数据序列化/反序列化
   - 数据导出功能

3. **UI控制模块 (UIController)**
   - 控制面板管理
   - 状态显示更新
   - 用户交互处理

4. **阅读界面模块 (ReadingView)**
   - 文章内容渲染
   - 阅读区域管理
   - 交互元素绑定
